# Project Principles

## Local First

Mojify should be useful with data stored in the user's browser profile. Remote services can enrich the library, but core browsing should stay fast after import.

## Honest Platform Support

Every chat site is different. Mojify should describe platform support truthfully and keep experimental adapters clearly labeled.

## Fast With Large Libraries

A user with thousands of emotes should still be able to open the popup, search, browse recent items, and switch scopes without waiting several seconds.

## Small Sharp Dependencies

Dependencies are welcome when they are worth their weight. The extension should avoid bundling large libraries unless they solve a hard browser-side problem.

## Friendly To Contributors

A bug report should have a clear template. A pull request should have obvious validation steps. A new contributor should understand the architecture without reading every line of `background.js` first.

## User Trust Beats Tricks

Mojify should avoid hidden network behavior, secret remote code loading, and unclear permission use. If a feature needs a powerful permission, the repo should explain why.
